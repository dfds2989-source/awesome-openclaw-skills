### 5. 创建 Issues 模板

**Bug 报告模板** (`.github/ISSUE_TEMPLATE/bug_report.md`)：

```markdown
---
name: Bug 报告
about: 报告网站或资源的问题
title: '[BUG] '
labels: 'bug'
assignees: ''
---

## 描述问题

清晰简洁地描述问题。

## 复现步骤

1. 访问...
2. 点击...
3. 看到...

## 预期行为

应该发生什么

## 实际行为

实际发生了什么

## 环境信息

- 浏览器：
- 操作系统：
- 其他：

## 截图

如果适用，请添加截图
```

**功能请求模板** (`.github/ISSUE_TEMPLATE/feature_request.md`)：

```markdown
---
name: 功能请求
about: 建议新功能或改进
title: '[FEATURE] '
labels: 'enhancement'
assignees: ''
---

## 功能描述

清晰描述您想要的功能。

## 解决的问题

这个功能解决什么问题？

## 建议的解决方案

您建议如何实现？

## 替代方案

是否有其他方式？

## 附加信息

其他相关信息
```

## 📊 GitHub 推广技巧

### 1. 优化 Commit 信息

**好的 Commit 信息格式：**
```
[类型] 简短描述

更详细的说明（可选）

- 具体改动 1
- 具体改动 2
```

**类型示例：**
- `[feat]` - 新功能
- `[fix]` - 修复 bug
- `[docs]` - 文档更新
- `[style]` - 代码风格
- `[refactor]` - 代码重构
- `[perf]` - 性能优化
- `[test]` - 测试相关

### 3. 添加 GitHub Actions

**自动化工作流示例** (`.github/workflows/ci.yml`)：

```yaml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: pnpm install
      - run: pnpm build
      - run: pnpm run check
```
