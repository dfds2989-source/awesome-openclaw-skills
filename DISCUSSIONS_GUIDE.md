# GitHub Discussions 社区指南

欢迎加入 OpenClaw 技能大全社区！我们使用 GitHub Discussions 作为主要的社区交流平台。

---

## 📢 讨论分类

### 1️⃣ 📣 公告 (Announcements)

**用途：** 项目更新、新版本发布、重要公告

**示例：**
- 新版本发布公告
- 重大功能更新
- 维护通知
- 社区活动公告

**发布者：** 项目维护者

---

### 2️⃣ 💬 问答 (Q&A)

**用途：** 提问和回答技能相关的问题

**示例：**
- 如何安装和使用某个技能？
- 技能 A 和技能 B 有什么区别？
- 如何为项目贡献新技能？
- 技能文档中的某个部分不清楚

**提问模板：**
```
## 问题
[清晰地描述您的问题]

## 背景
[提供相关的背景信息]

## 已尝试的方案
[列出您已经尝试的解决方案]

## 期望的结果
[描述您期望的结果]
```

---

### 3️⃣ 💡 想法 (Ideas)

**用途：** 提出新的想法和功能建议

**示例：**
- 建议添加新的技能分类
- 建议改进网站功能
- 建议改进用户体验
- 建议新的推广策略

**建议模板：**
```
## 想法标题
[简洁的想法标题]

## 问题描述
[描述这个想法解决的问题]

## 建议的解决方案
[描述您建议的解决方案]

## 替代方案
[描述任何替代方案或功能]

## 额外背景
[提供任何其他背景信息]
```

---

**示例：**
- 使用 OpenClaw 技能构建的项目

- 创意应用案例
- 成功故事

**分享模板：**
```

[项目或成果的名称]

[简要描述您的项目或成果]

## 使用的技能
```

## 📋 讨论礼仪

### ✅ 推荐做法

1. **保持尊重** - 尊重他人的观点和意见
2. **清晰表达** - 使用清晰的语言和结构
3. **搜索已有讨论** - 提问前先搜索是否有相似的讨论
4. **提供上下文** - 提供足够的背景信息
5. **使用代码块** - 分享代码时使用适当的格式
6. **及时回复** - 对他人的回复及时做出反应
7. **给予反馈** - 对有帮助的回复点赞或评论

### ❌ 避免做法

1. **垃圾信息** - 避免发布无关的内容
2. **自我推广** - 避免过度推广自己的产品或服务
3. **攻击他人** - 避免人身攻击或不尊重的语言
4. **重复提问** - 避免重复提出相同的问题
5. **发布敏感信息** - 避免分享密钥、密码等敏感信息
6. **长篇大论** - 保持讨论简洁明了
7. **离题讨论** - 保持讨论与主题相关

## 🎯 讨论最佳实践

### 提问时

1. **使用清晰的标题**
   - ✅ 好：如何在 React 中集成 ComfyUI？
   - ❌ 差：帮助！不知道怎么做

2. **提供具体的信息**
   - 使用的技能名称
   - 遇到的错误信息
   - 您的操作系统和版本
   - 已尝试的解决方案

3. **格式化您的内容**
   - 使用标题和列表
   - 使用代码块格式化代码
   - 使用引用突出重要信息

### 回答时

1. **直接回答问题**
   - 首先回答核心问题
   - 然后提供额外的上下文或建议

2. **提供有用的资源**
   - 链接到相关文档
   - 推荐相关技能
   - 分享有用的教程或指南

3. **验证您的答案**
   - 确保信息准确
   - 提供示例或演示
   - 邀请提问者反馈
